Module 01

Authentication & Context Switching — React & Azure Migration

A detailed technical design for transitioning the authentication, user locking, two-factor verification, and condominium context switching from legacy Yii2 / PHP backend sessions and database procedures to a modern React 19 SPA, stateless NestJS JWT-based architecture, and Azure Key Vault managed secrets.

Migration Blueprint

Module Summary

The current Authentication system runs on legacy Yii2 controllers using stateful PHP session cookies and invokes database stored procedures (e.g., admin_user_login_ver3) which perform decryption of PII fields (emails, usernames, and passwords) inside the DB using a shared, hardcoded private_db_key.

The proposed architecture moves to a stateless JWT-based model using a decoupled **React 19 SPA** (Vite-bundled) frontend on Azure Static Web Apps, and a **NestJS** backend API on Azure App Service. We integrate **Azure Key Vault** to handle security keys (like the JWT secret and database PII decryption keys) and utilize **Azure Redis Cache** to store session block lists, brute force counters, and active permissions to guarantee high availability, low latency, and maximum security.

🔐

Stateless JWT Auth

Short-lived Access Tokens in memory and Secure HttpOnly Refresh Tokens replace PHP session state cookies.

🔑

Azure Key Vault

Protects and dynamically resolves cryptographic secrets, API keys, and AES-256 PII decryption keys.

⚡

Azure Redis Cache

Performs rapid brute force rate-limiting, handles account lockout windows, and stores user permission profiles.

🔄

Context Switching

Enables Property Managers with multi-condo access to switch scopes cleanly via a switch-context API without re-authenticating.

Tenant Isolation & Scope Enforcement

JWT payloads will dynamically encode the user's active condoId. All downstream NestJS APIs intercept requests, extracting this claim to enforce strict per-condo data scoping, preventing cross-tenant data leaks.

Architecture & Actors

System Components

⚛️

React 19 SPA

Replaces legacy PHP Yii2 login views. Rich UI (TypeScript/Zustand), dynamic route guarding, secure session timers, and a seamless condominium context-switching dropdown.

🟢

Replaces Yii2 ApiController. Integrates passport-jwt, typed DTO validation, and custom decorators (@ActiveCondo()) for authorization.

🛡️

⚡

🗄️

📨

Migration Flow

Existing vs. Proposed — Step by Step

1
User credentials submission
The Property Manager inputs username/email and password on the portal login screen.
Legacy PHP / Yii2 Submitted via traditional form-data to POST /site/login. Credentials parsed and mapped to the Yii2 LoginForm model. No strict REST DTO contract.
Proposed React & NestJS React SPA calls POST /v1/auth/login with JSON body. NestJS validates schema via LoginDto (using class-validator and class-transformer).
2
Brute force & rate limit checks
The system checks for failed login attempts to prevent brute force security threats.
Legacy PHP / Yii2 Performs DB SELECT on user_login_time table. Adds record to user_otp_attempt and increments blocking states in user_blocked in MySQL. Writes to DB on every fail.
Proposed React & NestJS NestJS checks IP and email keys in **Azure Redis**. If failed attempts ≥ 5, locks account for 15 minutes. High-speed caching reduces DB overhead and protects MySQL/Azure Cosmos DB (NoSQL) from resource exhaustion.
3
Credentials validation & decryption
The system verifies credentials against DB records and decrypts encrypted PII fields.
Legacy PHP / Yii2 Calls SP admin_user_login_ver3. Decrypts email, phone, and display name inside MySQL query using a shared private_db_key passed from configuration files.
Proposed React & NestJS NestJS queries user record. Performs password verification (BCrypt/Argon2). Resolves decryption key from **Azure Key Vault** to decrypt PII at application layer, keeping database clean and secure.
4
Two-Factor Authentication (OTP verification)
If configured for the condominium, the system triggers a 2FA OTP code.
Legacy PHP / Yii2 Evaluates 2FA settings in condo_setting table inside MySQL, writes OTP to database, and uses a cron job to send OTP via SMS/Email queue.
Proposed React & NestJS NestJS checks 2FA requirements. Generates OTP, stores hash with 5-minute TTL in Redis. Dispatches notification event to Service Bus; `OTPWorker` sends email/SMS via **Azure Communication Services**.
5
JWT Generation & Role Mapping
On successful verification, the backend generates tokens and scopes user authority.
Legacy PHP / Yii2 Generates a PHP session ID and header token. Fetches permissions from common\models\Permission, storing HTML menus and action lists directly in Yii::$app->session.
Proposed React & NestJS Generates stateless **Access Token** (JWT, 15 min exp) containing basic user profile, scope, and allowed condominiums, and a secure **Refresh Token** (JWT, 7 day exp) stored in an `HttpOnly`, `Secure`, `SameSite=Strict` cookie.
6
Condo context switching
For Property Managers managing multiple condominiums, context can be toggled in the portal.
Legacy PHP / Yii2 Session variables (e.g., Yii::$app->session['permission'], Yii::$app->session['menu']) are mutated in place. Hard to verify context integrity between multiple browser tabs.
Proposed React & NestJS React dropdown triggers POST /v1/auth/switch-context with target condoId. NestJS verifies ownership and issues a new JWT Access Token scoped exclusively to the target condo. Clean, secure isolation.
7
Password reset & activation flows
Supports user forgot-password flow and first-time set-password for new accounts.
Legacy PHP / Yii2 Calls SP user_reset_password_ver4 or user_set_password_ver3. Sends raw code via the DB email queue. Token activation validation relies on SQL queries.
Proposed React & NestJS Forgot password endpoint generates a secure cryptographically random token with a 1-hour expiration stored in Redis. User clicks link, enters new password validated against password policies.
8
Logout & session invalidation
Destroys sessions and invalidates tokens on user exit.
Legacy PHP / Yii2 Destroys PHP Session via Yii::$app->user->logout(). Clears cookie variables on the response header.
Proposed React & NestJS React clears token from memory. Calls POST /v1/auth/logout. NestJS clears the HttpOnly Refresh Token cookie and blacklists the Access Token in Redis for its remaining time-to-live.

Security & RBAC

Security Controls and Authorization

Token Security & Expiration

Short Access Token lifespan (15 minutes) minimizes the exposure window of compromised tokens. Refresh tokens support rotation (re-issued on each refresh call) to detect and block token theft.

Role-Based Access Control (RBAC)

User roles (Property Manager, Condominium Admin, Staff) are mapped to allowed endpoints. NestJS RolesGuard intercepts calls, validating permissions matching the JWT active scope.

Distributed Session Revocation

Active logout blacklists the current JWT in Redis. If a user is deactivated or blocked, administrative events invalidate all associated refresh tokens immediately in the Azure Cosmos DB (NoSQL) session database.

Managed Identity Access

No database passwords or encryption keys are written in configuration files or code repositories. Azure App Service utilizes Managed Identity to authorize access directly to database and Key Vault resources.

Module 02

Dashboard Module

A pure frontend-driven architecture blueprint detailing the Property Manager metrics dashboard widgets, real-time list configurations, and responsive UI components using React 19 and Zustand.

Overview

Dashboard Widgets & UI Roster

The legacy Dashboard displays metrics compiled directly via blocking server-rendered Yii2 page layouts, resulting in high load times and rendering locks.

The proposed React 19 dashboard renders a **modular widget grid** resolving metrics asynchronously. All cards (e.g. Accounts, Facility Bookings, Overstaying Vehicles, General Feedback) run independent fetching promises and apply local caching to optimize performance.

📊

Asynchronous Widgets

Each dashboard metric is represented by an independent React widget card resolving states concurrently.

🔍

Dynamic Filtering

Enables instant directory filtering by condo blocks, floors, and user roles via client-side search states.

⏱️

Zustand State Store

Manages current active tabs, filter values, and loaded datasets in a lightweight client-side memory store.

💼

Account Lifecycle Actions

Empowers Property Managers to link, unlink, and trigger deactivations of resident and contractor accounts from the dashboard ledger.

Client-Side Layout Hydration

The dashboard uses dynamic CSS Grid configuration. Widgets adapt dynamically based on screen real estate, ensuring a seamless tablet and mobile browser administration portal.
